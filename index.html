SUPERSTORY INTERNAL — NEGOTIATION PREP / RED TEAM SIMULATION

치지직 요구안 시뮬레이션 — & 대응 매핑

Project AWAKE(각성) · 파트너십 협상 준비 문서 v0.1
치지직 사업 담당자의 관점에서 "30억을 태우는 대가로 내부 품의에 올릴 요구안"을 먼저 작성하고, 각 조항에 대한 우리의 대응 스탠스(수용 / 조건부 / 방어선)를 매핑한다. 목적: 우리 제안서가 일방적 '설득'이 아니라 상대 요구에 대한 '응답'이 되도록 — 상대가 원하는 것을 우리가 먼저 써봐야, 모두가 원하는 제안이 나온다. 본 문서는 순수 내부용 — 어떤 형태로도 외부 공유 금지.
01

치지직의 계산 — 그들이 사는 것, 그들이 두려워하는 것

구분내용
사는 것 ① 이적 불가능한 스타의 공급 파이프라인(캐릭터 IP 결속 구조 — 플랫폼 최초의 '떠날 수 없는 탤런트' 클래스) ② 국내외 버튜버 전용 스튜디오(기투자 캐펙스)의 가동률·기술 시연 ③ 시즌당 버튜버 버티컬 한 층(데뷔 5 + 비데뷔 19 + 탈락자 트랙) ④ 검증·팬덤 검증 완료 상태로 인도되는 탤런트(획득 리스크의 외주화) ⑤ 도네이션을 넘는 팬덤형 과금 레일 실증(젬·스탯·NFT).
두려워하는 것 ① 30억을 태우고 방송 한 편만 남는 것(시즌 후 자산 귀속 불명) ② 제3자 앱 리스크 — 네이버는 시스템상 외부 서드파티와 일하지 않음(7/6 확인), 예외를 뚫으려면 임원 보고용 명분 필요 ③ 투표 조작 시비가 네이버 브랜드에 튀는 것(프로듀스 트라우마) ④ 데뷔 시점(노출 최대치)에 리스크를 인수하는 구조 ⑤ IP는 남의 것인데 비용은 자기가 다 내는 그림 — "잘되면 슈퍼스토리 IP 가치가 오르고, 못되면 우리 손실" ⑥ 편성·운영 통제권 없이 자기 플랫폼 간판을 거는 것.
내부 사정 정해진 버짓 없음 — 경영 보고부터 새로 받는 구조(7/6). 즉 담당자는 "임원을 설득할 수 있는 요구안"이 필요하다: 금액이 클수록 리턴 조항이 촘촘해야 품의가 통과된다. 우리 제안서는 사실상 그의 품의서 초안이 되어줘야 한다.
02

치지직 요구안 (시뮬레이션 텀시트)

아래는 치지직 담당자가 30억 규모 품의에 붙일 법한 요구 조항을 6개 카테고리로 재구성한 것이다. ★ = 그쪽에서 절대 물러서기 어려운 조항(임원 보고 통과 조건)으로 추정.

카테고리요구 조항
A. 투자 회수 A1 ★ 우선 회수(recoupment) — 시즌 관련 수익(젬 RS·음원·굿즈)에서 제작비 회수 전까지 치지직 우선 배분. A2 앱 내 결제 수익 셰어 — "트래픽은 우리가 보낸다": 방송 연동 기간 젬 매출의 일정 지분. A3 마일스톤 분할 지급(기획 확정 / 앱 출시 / 시즌 개시 / 데뷔) + 핵심 지표 미달 시 후속 지급 조정. A4 데뷔조 2차 사업(음원·굿즈·광고) 수익 지분.
B. 독점권 B1 ★ AWAKE 세계관 라이브 방송 치지직 독점(계약 기간, 비데뷔 19인 포함). B2 ★ 데뷔조 5인 장기 전속(방송 + 매니지먼트, 최소 N년). B3 경업 금지 — 슈퍼스토리·LVNEXUS는 동일 포맷을 계약 기간 내 타 플랫폼(SOOP·유튜브 라이브 등)과 진행 불가. B4 차기 시즌 + 글로벌 시즌(해외 스튜디오 활용) 우선협상권.
C. IP 안전장치 C1 데뷔조 캐릭터 IP 양도 또는 공동 소유(1차 요구 — "비용을 전부 대는데 자산이 남의 것일 수는 없다"). C2 ★ 하자담보 — 인계 후 X개월 내 G2 이전 사유 발각 시 슈퍼스토리 책임(배상 또는 대체). C3 소울 귀책 졸업 시 보전 — 라이선스료 감액 또는 대체 캐릭터. C4 헌법 준수 조항의 상호성 — 슈퍼스토리 귀책(IP 분쟁·디자인 표절 시비 등) 시 동등한 배상 구조.
D. 리스크·브랜드 D1 ★ 투표 무결성 감사권 — 제3자 감사 또는 치지직 감사권 + 원장 접근(프로듀스 사태가 네이버 브랜드에 튀는 것 원천 차단). D2 ★ 위기 통보·협의권 — 실격·폭로 등 중대 사안의 치지직 지정 창구 즉시 통보 + 공동 대응. D3 검증 품질 진술보증(rep & warranty) — "검증했다"의 계약적 보증. D4 네이버 플랫폼 정책·미성년 보호 정책 준수 증빙(청소년보호책임자·시간 원장 등). D5 앱 장애 SLA — 생방 중 투표 앱 다운 = 방송 사고: 가동률 보증 + 페널티.
E. 운영 통제 E1 ★ 편성 최종 결정권 — 자기 스튜디오·트래픽 최적화 기준. E2 공동 운영위원회(주요 의사결정 협의체) + 제작 크레딧 "치지직 스튜디오 제작" 브랜딩. E3 시청·투표 데이터 공유(네이버는 데이터 회사다). E4 기존 치지직 스트리머·버튜버의 참여 통로(7/6 리더 요구 — 생태계 부스팅 명분).
F. 명분(대내) F1 제3자 앱 예외 승인을 위한 명분 패키지 — "제작비 절감 + 콘텐츠 영역을 앱이 채운다"(7/6 긍정 반응)의 문서화. F2 "30억 → 받는 것" 1페이지 계산표(임원 보고용) — 우리가 만들어서 줘야 하는 것.
03

대응 매핑 — 수용 / 조건부 / 방어선

조항스탠스대응
B1 라이브 독점즉시 수용어차피 치지직에서 한다 — 비용 제로, 상대 체감 가치 최대. 선제 제안 카드 1순위("플랫폼 영구 홈 조항").
D1 무결성 감사즉시 수용미결 ⑩에서 이미 "원장 공개 = 신뢰 메시지"로 설계됨. 제3자 감사 수용이 오히려 우리 셀링 포인트.
D2 위기 통보즉시 수용레드팀 지적(P1-③)과 동일 — 통합본 §13에 파트너 에스컬레이션 조항으로 반영 예정.
D4 정책 준수 · E4 스트리머 참여 · F 명분 패키지즉시 수용D4는 통합본 §11이 이미 답. E4는 "전생 공개 자유 + 별도 무대"로 설계 완료. F2 계산표는 우리가 먼저 만들어 간다 — 상대의 품의서를 우리가 써주는 것이 최고의 협상.
A1 우선 회수조건부수용하되 범위 한정: 시즌 직접 수익 내 회수(캡 설정), 앱 인프라 투자분은 제외. 회수 후 배분율은 별도 협상.
A2 젬 셰어조건부방송 연동 구간(라이브 중 투표 등 트래픽 기여 명확 구간)의 매출 셰어는 수용 — 단 앱 전체 매출 셰어는 불가(7/6 지분 방어선: 결제·아이템·채팅 RS는 우리 심장).
B2 데뷔조 전속조건부장기 독점 라이선스로 수용(경제 효과는 보유와 동등) — 소유 아닌 대여. 티어형 전속(참가규정 §03)과 정합.
B3 경업 금지조건부동일 포맷·계약 기간·한국 시장 한정으로 수용. 글로벌은 B4 우선협상권과 교환 — 영구·전 영역 경업 금지는 불가.
C2 하자담보조건부수용하되 범위 한정: G2 검증 범위 내 + 공적 기록 기준(§09 기준표) + 기간 캡(예: 인계 후 6개월). 검증 불가능 영역(사적 폭로류)은 면책 — 검증 프로토콜 문서가 그 경계의 증빙.
C3 귀책 보전 · C4 상호성조건부C3 = 대체 캐릭터 라이선스로 수용(재캐스팅 금지와 양립하는 유일한 보전). C4 상호성은 공정하므로 수용 — 우리 귀책 정의를 명확히.
D5 SLA · E3 데이터조건부SLA는 수용(수준·페널티 협상, M4 무결성 설계가 뒷받침). 데이터는 집계·비식별 데이터 공유는 수용, 유저 원천 데이터는 불가(개인정보 + 앱 자산).
E1 편성권조건부편성 결정권은 존중하되 세계관 제작 요구사항을 편성 전제 조건으로 계약에 명시: 검증 최소 기간(결선 X주·확정~각성 Y일), 미성년 22시 컷, 각성 모먼트 배치 협의권(미결 ⑪ 연동). "당신이 시간표를 짜되, 이 제약은 물리 법칙이다."
C1 IP 양도방어선원본 양도·공동 소유 불가. 대체 패키지로 응답: B1 영구 홈 + B2 장기 독점 라이선스 + A4 2차 사업 지분 + C3 보전 — "소유가 주는 경제 효과 전부를 소유 없이 드린다." 헌법 준수를 라이선스 존속 조건으로 명시(위반 시 회수권 = 우리의 무기). 최후 카드(먼저 꺼내지 않음): 데뷔조 한정 공동 지분 — 헌법 준수를 지분 조건으로.
비데뷔 19인 · 차기 IP방어선파이프라인의 소유 — 협상 대상 아님. 19인의 라이브 독점(B1)으로 치지직 이익은 충족된다.
04

협상 설계 — 기브 & 테이크 요약

먼저 주는 것 (선제 제안)받는 것지키는 것
라이브 영구 홈(B1) · 무결성 감사권(D1) · 위기 통보(D2) · 명분 패키지(F) · 데뷔조 장기 독점 라이선스(B2) 30억 제작·기술비 · 스튜디오 · 트래픽·프로모션 · 제3자 앱 예외 승인 · 글로벌 시즌 파트너십 캐릭터 원본 IP · 헌법 해석권 · 앱(결제·아이템·채팅 RS) · 비데뷔 19인 · 차기 시즌
제안서 반영 원칙
① 우리 제안서의 목차를 이 요구안에 1:1로 응답하는 구조로 짠다 — 상대 담당자가 그대로 품의에 붙일 수 있게. ② "30억 → 받는 것" 1페이지 계산표(F2)를 우리가 만들어 간다. ③ IP 논의는 우리가 먼저 꺼낸다 — "양도는 없지만, 소유의 경제 효과는 전부 드린다"를 선제 프레임으로(상대가 C1을 꺼낸 후 방어하면 늦다). ④ 통합본(설계헌법·참가규정 v0.2)의 검증·보호 프로토콜은 협상 자료로 요약 발췌해 제시 — "우리는 리스크에 돈을 쓰는 파트너"가 D 카테고리 전체의 답이다.
